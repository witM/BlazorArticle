Installation

The library is designed to work with .NET and Blazor applications. You can choose either a Blazor Web App or a Blazor WebAssembly Standalone application. For example, in a Blazor Web App, you might render articles from a database and build a custom article editing system. For simplicity, you can use a Blazor WebAssembly Standalone application for editing articles and then push them to the database to make them publicly visible. This is how the documentation is written.

Package installation

Install the following packages from nuget.

Console 

dotnet add package BlazorArticle
dotnet add package BlazorArticle.Components

In the Visual Studio gui package manager look for "BlazorAricle" and "BlazorArticle.Components" packages.

After the packages are installed, add the following code to the Program.cs file to use the library's default services.

Program.cs 

builder.Services.AddBlazorArticle();
builder.Services.AddBlazorArticleStyle();
//optional
builder.Services.AddBlazorArticleDefaultMarkers();

Assets

When using the BlazorArticle.Components library, you need to include the following assets according to the project type — either Blazor Web App or Blazor WebAssembly Standalone.

# 

<link rel="stylesheet" href="_content/BlazorArticle.Components/css/blazorArticle.css" />
# 

<script src="_content/BlazorArticle.Components/js/blazorArticle.js"></script>

The styles from the blazorArticle.css file start with the prefix "m-".
